# 基于文献计量学的 LLM 多智能体协作研究前沿追踪：Mini Review 初稿

## 摘要

大语言模型正在推动人工智能系统从单轮文本生成转向具备任务分解、工具调用、协作规划和自主执行能力的智能体系统。尤其在复杂工程、软件开发、机器人、人机交互和知识管理等场景中，多个 LLM 智能体之间的协作机制逐渐成为新的研究热点。本文基于 Lens.org 学术数据库构建了 LLM 多智能体协作领域的文献数据集，经过 DOI 与标题去重后获得 522 篇有效文献。研究采用文献计量学与科学知识图谱方法，分析年度发文趋势、被引影响力、关键词共现、作者合作网络、机构/国家分布和参数敏感性。结果显示，该领域在 2024-2025 年进入爆发式增长阶段，2025 年发文量达到 344 篇；总被引次数为 4,265 次，h 指数为 31；关键词网络显示研究核心集中于 artificial intelligence、large language models、agentic AI、multi-agent systems、retrieval-augmented generation 和 ethics 等主题。作者合作网络显示该领域仍处于快速扩张和团队分散阶段，尚未形成高度稳定的核心作者群。本文进一步讨论了 LLM 多智能体研究的主要趋势、研究空白与未来方向，并给出可复现的项目流水线。

**关键词**：文献计量学；大语言模型；多智能体系统；科学知识图谱；前沿趋势追踪；关键词共现

## 1. 引言

大语言模型（Large Language Models, LLMs）的能力提升正在改变人工智能系统的组织方式。早期 LLM 应用多集中于单模型问答、文本生成和信息抽取；随着工具调用、检索增强生成、长上下文和代码生成能力的发展，LLM 开始被组织为能够分工、协作、反思和迭代的智能体系统。多智能体协作由此成为理解新一代 AI 系统的重要入口：它不仅关注模型本身能力，也关注智能体之间的角色分配、通信协议、任务规划、记忆管理、错误恢复与人类监督。

从课程项目的要求看，文献计量综述不能停留在资料堆叠，而需要用可复现的数据、图谱和指标证明"为什么这些方向值得讨论"。因此，本研究选择 LLM 多智能体协作作为对象，尝试回答四个问题：

1. LLM 多智能体研究在时间上如何演化，是否已经进入爆发期？
2. 该领域的核心主题和关键词结构是什么？
3. 哪些作者、机构或国家正在推动该领域发展？
4. 当前研究存在哪些方法、应用和评估层面的空白？

本文的贡献有三点。第一，构建了固定检索式、固定时间窗和固定数据版本的可复现数据集。第二，形成了覆盖 M1/M2/M3 的分析流水线，能够一键生成主要图表与统计报告。第三，将计量结果转化为综述叙事，讨论 LLM 多智能体协作从通用智能体框架走向垂直工程应用的趋势。

## 2. 数据与方法

### 2.1 数据来源与检索策略

本研究数据来自 Lens.org Scholarly Works。检索对象聚焦于 LLM 与多智能体协作的交叉领域，核心检索逻辑采用 `(Object) AND (Method)` 的布尔组合。Object 侧包括 `Large Language Model*`、`LLM*`、`Generative AI`、`Foundation Model*`、`ChatGPT` 等术语；Method 侧包括 `Multi-agent*`、`Collaboration`、`Coordination`、`Cooperation`、`Multi-agent system*` 等术语。时间范围设定为 2020-2026 年，文献类型包括 Article、Review 与 Conference Paper，语言限定为 English。

原始导出记录为 542 篇。项目使用 `src/data_quality.py` 对数据进行清洗，先按 DOI 去重，再按 Title 去重，最终得到 522 篇有效文献。核心字段完整度较高：Title 与 Publication Year 覆盖率为 100%，Abstract 覆盖率为 98.28%，References 覆盖率为 81.99%。该数据质量能够支持发文趋势、关键词共现、作者合作和引用网络等分析。

### 2.2 分析工具与可复现流程

本项目主要使用 Python 进行数据处理、指标计算和图谱绘制，核心依赖包括 `pandas`、`numpy`、`networkx`、`matplotlib`、`seaborn`、`scipy` 和 `wordcloud`。仓库新增 `requirements.txt` 固定依赖范围，并新增 `run_pipeline.py` 作为一键运行入口。复现命令如下：

```bash
pip install -r requirements.txt
python run_pipeline.py
```

流水线依次执行数据质检、发文趋势图、标题词云、计量指标、关键词共现网络、作者合作网络、机构/国家分布、引用网络与敏感性分析。输出文件主要保存在 `outputs/`、`outputs/figures/` 与 `reports/` 中。

### 2.3 指标与图谱方法

计量指标包括年度发文量、年增长率、总被引次数、篇均被引、被引中位数、高被引论文数、零被引论文数、h 指数、来源期刊分布、文献类型分布、开放获取比例、作者统计和研究领域分布。

关键词共现网络以同一篇论文中共同出现的关键词为连边依据。为避免重复关键词造成自连边，本文在单篇文献内先对关键词进行标准化和去重，再统计共现关系。作者合作网络以共同署名为连边依据，并过滤无效作者名，如 `null`、`null null`、`nan` 和 `unknown`。参数敏感性分析覆盖关键词共现阈值、作者合作阈值、时间窗口和节点度过滤四类设置。

## 3. 结果

### 3.1 发文趋势：2024-2025 年进入快速爆发期

522 篇有效文献的年度分布显示，LLM 多智能体研究呈现明显的"萌芽-扩张-爆发"阶段。2020 年仅 1 篇，2021 年 2 篇，2022 年 6 篇，2023 年 19 篇，2024 年增长至 81 篇，2025 年进一步跃升至 344 篇。2026 年已有 69 篇，但由于数据采集时 2026 年尚未结束，该年份不能直接与完整年份比较。

这一趋势说明，多智能体协作研究并非传统多智能体系统的简单延续，而是在大模型能力跃迁之后快速形成的新研究聚合点。2024-2025 年的爆发可能与 GPT-4、Claude、Gemini 等通用模型能力提升，AutoGPT、MetaGPT、ChatDev 等智能体框架传播，以及工具调用、RAG 和代码执行环境成熟有关。不过，发文量增长反映的是学术关注度与研究活动强度，并不等同于技术已经成熟落地。

### 3.2 影响力指标：高增长与高稀释并存

样本总被引次数为 4,265 次，篇均被引 8.17 次，被引中位数为 1.00 次，h 指数为 31。高被引论文数为 20 篇，零被引论文数为 236 篇，占比约 45.2%。这一结构显示出新兴领域常见的双重特征：一方面，少数早期代表性论文和框架论文快速积累引用，形成知识锚点；另一方面，大量 2024-2025 年新论文尚未经过充分引用周期，导致中位数较低、零被引比例较高。

来源期刊/会议方面，International Joint Conference on Autonomous Agents and Multiagent Systems、Frontiers in Robotics and AI、Frontiers in Artificial Intelligence、Scientific Reports 和 Sensors 等位居前列。文献类型以期刊论文为主，占 73.4%；会议论文占 26.6%。开放获取比例为 73.18%，说明该领域知识传播壁垒相对较低，也有利于快速复现和二次开发。

### 3.3 关键词共现：从 LLM 核心能力走向 agentic AI 与应用治理

关键词统计显示，最高频关键词包括 `artificial intelligence`、`large language models`、`large language model`、`machine learning`、`natural language processing`、`deep learning`、`generative ai`、`agentic ai`、`chatgpt`、`reinforcement learning`、`ethics`、`multi-agent systems`、`retrieval-augmented generation` 和 `ai agents`。

这些关键词可以归纳为四个层次。第一是技术底座层，以 AI、LLM、machine learning 和 NLP 为核心，说明研究仍以通用人工智能能力为基础。第二是智能体架构层，包括 agentic AI、multi-agent systems、AI agents 和 coordination，体现研究从单模型能力转向系统组织机制。第三是能力增强层，包括 RAG、reinforcement learning、planning 与 workflow，代表智能体系统在知识获取、决策优化和任务执行方面的增强路径。第四是治理与应用层，包括 ethics、human-robot interaction、digital health 等，说明多智能体系统正在进入需要安全、责任和人类监督的实际场景。

关键词共现阈值敏感性显示，当阈值为 1 时网络边数过多，噪声较大；阈值为 2 时仍保留 233 个左右节点与 2,500 条左右边，能够兼顾信息量与可读性；阈值升至 5 或 10 后，网络迅速收缩，容易遗漏长尾主题。因此，阈值 2 更适合作为主要解释口径。

### 3.4 作者合作与机构分布：研究力量分散，核心团队尚未稳定

作者统计显示，样本涉及约 2,565 名作者，篇均作者数约 5.10 人。高产作者最高发文量仅为 3 篇，说明该领域尚未形成高度集中的核心作者群。作者合作网络原始规模较大，但当过滤条件设为合作次数不少于 2 次时，仅保留约 51 个节点和 48 条边。这说明多数合作关系仍是一篇论文层面的临时合作，小团队与新进入者较多，跨团队稳定合作网络尚未充分形成。

机构/国家分布显示，美国、英国、德国、瑞士、中国、韩国、日本、澳大利亚等国家均有研究贡献。该领域具有明显的国际化和跨学科特征，既连接计算机科学与人工智能，也延伸到机器人、医学、心理学、知识管理和工程系统。

## 4. 讨论

### 4.1 领域阶段判断

综合发文趋势、被引结构和合作网络，LLM 多智能体协作领域目前处于高速成长期，而非成熟稳定期。高速增长说明研究问题具有显著吸引力；作者和团队分散说明领域边界仍在形成；零被引比例较高说明很多成果尚未经过时间检验。对于综述写作而言，这意味着不能只按高频关键词罗列热点，还需要区分"当前关注度高"与"已形成可靠知识基础"。

### 4.2 主要研究空白

第一，标准化评估体系不足。多智能体系统常被用于复杂任务，但不同论文的任务设置、智能体数量、工具环境、记忆机制和评价指标差异较大，导致结果难以横向比较。

第二，协作机制解释性不足。许多研究展示多智能体框架能够完成任务，却缺少对通信、分工、冲突解决、失败恢复和人类监督机制的细粒度分析。

第三，垂直领域验证不足。LLM 多智能体在软件开发、硬件 EDA、RTL/Verilog 代码生成、医学辅助和机器人等领域具有潜力，但这些场景对可靠性、可验证性、安全边界和责任归属要求更高，不能简单套用通用聊天任务的评价方式。

第四，安全与伦理问题更加复杂。多智能体系统中的错误可能通过智能体间通信被放大，也可能因自动工具调用而产生外部影响。因此，伦理、对齐、可控性和审计机制应成为该领域的重要研究方向。

### 4.3 未来方向

未来研究可以从四个方向推进。第一，建立面向多智能体协作的标准 benchmark，覆盖任务成功率、成本、鲁棒性、可解释性和安全性。第二，发展更透明的协作协议，使智能体之间的消息、决策与工具调用可以被追踪和审计。第三，面向垂直工程场景开展高质量实证研究，特别是代码生成、EDA、科学发现和医疗工作流等高风险领域。第四，将文献计量图谱与人工精读结合，识别真正具有里程碑意义的框架、方法和应用案例。

## 5. 局限性

本研究仍存在若干局限。第一，数据仅来自 Lens.org，未与 Web of Science、Scopus、CNKI 等数据库进行交叉验证，可能存在数据库覆盖偏差。第二，关键词依赖数据库导出字段，可能受到作者关键词质量和平台标准化规则影响。第三，引用网络受 References 字段完整度限制，当前更适合做初步知识传播观察，不宜过度解释为完整共被引结构。第四，本文主要使用 Python/NetworkX 进行图谱构建，尚未充分利用 CiteSpace 的突现检测、共被引聚类和结构变异分析，这也是后续可加强的方向。

## 6. 结论

本文基于 522 篇有效文献，对 LLM 多智能体协作研究进行了可复现的文献计量分析。结果表明，该领域在 2024-2025 年进入快速爆发期，总被引次数和 h 指数显示其已形成初步影响力，但作者合作网络和引用分布也显示领域仍处于扩张与分化阶段。关键词共现揭示出技术底座、智能体架构、能力增强和治理应用四个层次。未来研究需要从标准化评估、协作机制解释、垂直领域验证和安全治理四方面深入推进。

从课程项目角度看，本项目已经形成从检索式、数据清洗、指标计算、图谱输出到 mini review 写作的完整证据链，并提供 `python run_pipeline.py` 作为一键复现入口。后续若补充 CiteSpace/VOSviewer 的共被引聚类和突现词分析，将进一步增强 milestone 论文识别和趋势判断的可信度。
