# AI 使用说明

## 1. 使用目的

本项目允许使用 AI 工具进行辅助，但 AI 不作为事实来源，不替代人工判断，不编造引用、数据和结论。所有关键数值、图表和论文信息均来自本项目的 Lens.org 数据、Python 脚本输出、阶段报告和人工核验。

## 2. 使用范围

| 使用场景 | 是否使用 | 说明 |
|---|---|---|
| README 结构整理 | 是 | 辅助检查 GitHub 首页是否覆盖研究主题、成员分工、数据来源、运行命令和输出目录 |
| 文字润色 | 是 | 辅助优化中文表达和段落逻辑 |
| 图表解释逻辑检查 | 是 | 辅助检查图表是否对应 RQ、是否存在“为了插图而插图”的问题 |
| 代码问题排查 | 是 | 辅助定位脚本运行、路径和依赖问题 |
| 数据结论生成 | 否 | 发文量、被引次数、h 指数、字段覆盖率等均来自本地数据和脚本 |
| 引用或 DOI 生成 | 否 | 高被引文献、DOI 和来源均来自 `outputs/milestone_paper_candidates.md` |

## 3. 人工核验方式

本项目对 AI 辅助内容采用以下核验方式：

1. 数值核验：所有关键指标回到 `outputs/metrics_summary.txt`、`reports/m2_metrics_analysis_report.md` 或脚本输出。
2. 图表核验：所有 README 图片路径均指向 `outputs/figures/` 或 `outputs/review_figures/` 中的本地文件。
3. 文献核验：Top 10 高被引文献表来自 `outputs/milestone_paper_candidates.md`。
4. 方法核验：检索式、时间范围、文献类型和语言限制回到 `config/query.yaml` 与 `data/README.md`。
5. 结论核验：Results 和 Discussion 中的判断均尽量附带边界说明，避免从文献计量结果过度推出技术成熟度或产业效果。

## 4. 使用边界

AI 不用于：

- 编造不存在的参考文献。
- 编造 DOI、期刊、作者或被引次数。
- 替代 Lens.org 数据检索。
- 替代人工判断研究问题和结论边界。
- 生成无法回到项目文件核验的数据结论。

## 5. 规范声明

本项目中的 AI 使用属于辅助写作、代码排查和逻辑检查。最终提交内容由小组人工检查和确认，所有核心数据、图表、文献和结论均以项目仓库内可追溯材料为准。
