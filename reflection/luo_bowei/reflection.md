# 罗博伟个人收获与反思

## 1. 我在项目中的工作

在本项目中，我主要参与文献筛选、代表文献梳理和 mini review 写作相关内容。我的工作集中在 M3 阶段，主要围绕 milestone 候选论文、技术叙事线和未来挑战展开。

我参与的主要工作包括：

1. 协助整理高被引 milestone 候选论文。
2. 参与 mini review 中技术主线的讨论。
3. 协助归纳多智能体任务规划、科学发现、人机协作、医疗和机器人等方向。
4. 参与未来挑战和研究空白部分的文字讨论。
5. 为 M3 综述终稿提供部分内容基础。

项目后期的 README 重写、表格补齐、反思目录、AI 使用说明、参数说明和最终归档主要由高俊杰完成。

## 2. 我的收获

通过这个项目，我对文献综述写作有了更实际的理解。以前我理解的综述可能是把很多论文放在一起介绍，但课程和项目让我认识到，Review 的重点不是罗列文献，而是归纳主题、解释结构和形成判断。

在整理 milestone 候选论文时，我发现高被引论文可以帮助我们找到领域中更重要的研究方向。例如，自驱动实验室、机器人任务规划、临床智能体和人机协作这些方向，都可以从高被引论文中找到线索。但高被引不等于绝对正确，也不等于这个方向已经成熟，这一点需要在写作时保持谨慎。

我也学习到，综述写作需要和图表证据结合。不能先写一个目录，再随便找图放进去，而应该让关键词图谱、高被引论文表和共被引网络共同支撑技术主线。

## 3. 我的反思

我觉得自己在综述写作方面有参与，但对文献计量证据和 Results 段落之间的关系理解还可以更深入。最开始我更习惯写背景介绍和方向概括，但对“这句话由哪张图支持”“这条结论的边界是什么”考虑不够。

后期项目中，高俊杰对 README、mini review 和图表逻辑进行了较多整合，让我意识到综述写作不能只靠文字表达，还要和数据、图表、代表文献形成证据链。未来我需要更主动地把文献阅读转化为 evidence table，而不是只停留在段落写作。

## 4. 未来可以学习的内容

后续我希望继续学习：

1. 如何阅读和整理代表性高被引论文。
2. 如何制作 evidence table，将论文、主题、方法、贡献和局限对应起来。
3. 如何写更规范的 Results 和 Discussion。
4. 如何避免过度推断，尤其是避免把文献关注度直接写成技术成熟度。
5. 如何使用 Zotero 或 EndNote 管理参考文献，提高综述写作效率。

## 5. 总结

这次项目让我认识到，mini review 的价值不在于列出多少论文，而在于能否把图表证据和代表文献组织成清晰的技术叙事。我在项目中主要参与了文献和综述部分，未来还需要加强证据表整理和方法透明度意识。
