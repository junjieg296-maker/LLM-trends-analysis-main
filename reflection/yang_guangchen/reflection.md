# 杨广宸个人收获与反思

## 1. 我在项目中的工作

在本项目中，我主要参与 M2 阶段的计量分析和知识图谱构建。我的工作重点是协助完成年度趋势、关键词共现、作者合作、机构 / 国家分布、引用网络和共被引网络等图表输出。

我参与的主要工作包括：

1. 协助进行基础计量指标分析。
2. 参与关键词共现网络构建和图表输出。
3. 参与作者合作网络、机构 / 国家分布网络和引用相关网络的构建。
4. 协助完成参数敏感性分析。
5. 为 M2 报告和后续 mini review 提供图表基础。

项目后期关于图表如何对应 RQ、README 如何表达、3 图 1 表如何整理，以及最终 GitHub 归档，主要由高俊杰完成。

## 2. 我的收获

通过这个项目，我对文献计量图谱有了更具体的认识。以前我看到关键词共现图、作者合作图时，更多关注图是否好看。但这次项目让我意识到，图表最重要的是回答问题。

例如，关键词共现网络可以帮助识别研究热点，但不能直接说明某个方向一定成熟；作者合作网络可以显示合作结构，但不能直接评价谁的研究质量最高；共被引网络可以反映知识基础，但不能说明被引高的论文一定正确。

我也学习到参数对图表影响很大。关键词共现阈值如果太低，网络会非常复杂，噪声很多；阈值太高，又会丢失信息。通过敏感性分析，我理解了为什么项目选择共现频次大于等于 2 作为相对平衡的阈值。

## 3. 我的反思

我觉得自己在图表生成方面有一定参与，但对图表解释和论文写作的连接还不够。最开始我更关注“能不能生成图”，而不是“这张图回答哪个 RQ”“图中的节点、颜色、连线分别代表什么”“这张图不能推出什么”。

后期高俊杰对 README 和综述中的图表逻辑进行了大量整理，我也从中意识到，文献计量分析不能停留在技术操作层面。图表必须进入 Results 段落，并通过 Claim-Evidence-Interpretation-Boundary 的方式被解释。

## 4. 未来可以学习的内容

后续我希望继续学习：

1. NetworkX 的网络指标，例如 degree、centrality、community detection。
2. CiteSpace 和 VOSviewer 的专业图谱分析方法。
3. 如何为每张图写规范图注，包括数据来源、时间范围、工具、参数和视觉元素含义。
4. 如何做更系统的参数敏感性分析。
5. 如何把图表结果转化为论文中的 Results 段落。

## 5. 总结

这次项目让我认识到，图表生成只是文献计量分析的第一步，更重要的是解释图表、限定结论边界，并让图表服务于 RQ。我在项目中主要参与了 M2 图表和计量分析，未来还需要提高图表解释和学术写作能力。
