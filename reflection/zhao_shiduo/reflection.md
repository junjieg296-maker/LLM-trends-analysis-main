# 赵世铎个人收获与反思

## 1. 我在项目中的工作

在本项目中，我主要参与的是前期数据获取和检索策略相关工作。我负责协助确定 Lens.org 的检索口径，围绕 LLM 多智能体协作这一主题，参与讨论 Object 和 Method 两组关键词的设置。

我参与的工作主要包括：

1. 协助确定检索对象词组，例如 `Large Language Model*`、`LLM*`、`Generative AI`、`Foundation Model*` 和 `ChatGPT`。
2. 协助确定方法词组，例如 `Multi-agent*`、`Collaboration`、`Coordination`、`Cooperation` 和 `Multi-agent system*`。
3. 参与 Lens.org 原始数据导出，形成 542 条初始记录。
4. 协助记录检索口径和数据来源，为后续清洗和计量分析提供基础。

我的工作主要集中在项目早期，后续 README、代码整合、图表解释、最终归档等大量工作主要由高俊杰完成。

## 2. 我的收获

通过这次项目，我对文献计量学中“检索式”的重要性有了更深理解。以前我可能觉得检索就是输入几个关键词，但实际做项目时发现，检索式会直接影响后续所有分析结果。

如果检索式太宽，样本中会混入很多不相关文献，后续关键词共现和主题分析就会跑偏。如果检索式太窄，又可能漏掉重要文献，导致综述不全面。因此，检索式需要在查全率和查准率之间取得平衡。

我也认识到，课程中强调的 Query as Code 很重要。把检索式写进 `config/query.yaml`，比只在口头上说“我们检索了 LLM 和 multi-agent”更可靠。这样别人可以复查我们的检索逻辑，也可以根据同样的口径重新获取相近数据。

## 3. 我的反思

我觉得自己在这个项目中的参与主要集中在前期，后期对图表解释、README 整理和最终材料归档参与不够。尤其是课程后期要求不仅要有数据，还要有 RQ、3 图 1 表、Methods、AI 使用说明和 reflection，我对这些材料的主动参与还不够。

如果以后再做类似项目，我需要更早地把自己的检索工作和后续分析连接起来。例如，我不仅要提供检索式，还应该说明为什么选择这些关键词、可能漏掉哪些文献、哪些检索词可能带来噪声，以及检索口径如何影响结果解释。

## 4. 未来可以学习的内容

后续我希望重点学习以下内容：

1. 如何设计更专业的布尔检索式，包括字段限定、同义词扩展和排除词设置。
2. 如何比较不同数据库，例如 Lens.org、Web of Science、Scopus 和 CNKI。
3. 如何记录检索日期、检索版本和筛选理由，使数据来源更加透明。
4. 如何把检索策略写进 Methods，而不是只停留在操作层面。
5. 如何参与后续的图表解释和综述写作，让前期检索工作更好地服务最终论文。

## 5. 总结

这次项目让我认识到，数据获取不是简单下载 CSV，而是整个文献计量项目的起点。我的主要贡献是协助完成前期检索和数据获取，但我也意识到自己后期参与不够。未来我需要提高从检索设计到结果解释的完整项目意识。
