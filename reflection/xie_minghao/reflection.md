# 解明昊个人收获与反思

## 1. 我在项目中的工作

在本项目中，我主要参与数据清洗和质量控制相关工作。我的任务集中在 M1 阶段，帮助处理 Lens.org 导出的原始数据，并参与 DOI 与 Title 去重、字段缺失率检查和清洗数据输出。

我参与的主要工作包括：

1. 协助检查原始数据字段是否完整。
2. 参与 DOI 去重和 Title 去重逻辑。
3. 协助确认原始 542 条记录清洗后得到 522 条有效文献。
4. 参与 M1 数据质量报告相关内容。
5. 为后续 M2 指标计算、关键词共现和引用网络提供清洗后的数据基础。

项目后期的 README 完善、复现入口、图表逻辑修订和最终归档主要由高俊杰完成。

## 2. 我的收获

通过本项目，我认识到数据清洗在文献计量分析中非常重要。之前我可能更关注最后生成的图表，但这次项目让我意识到，如果底层数据有重复、字段缺失严重或格式不统一，后面的发文趋势、作者合作网络和关键词共现网络都会受到影响。

我印象比较深的是 DOI 和 Title 双重去重。单独依靠 DOI 可能会漏掉 DOI 缺失或格式不一致的重复记录；而单独依靠 Title 又可能遇到标题大小写、标点或格式差异。因此，去重需要结合多个字段，并且要在报告中说明清楚。

我还学到，字段覆盖率本身也是一种结果。比如 Title 和 Year 覆盖率为 100%，说明年度趋势和标题分析比较可靠；References 覆盖率为 81.99%，说明可以做引用和共被引分析，但解释时需要说明边界。

## 3. 我的反思

我觉得自己对数据清洗的参与还比较偏操作层面，对清洗规则背后的方法意义理解还不够深入。比如我知道要去重，但一开始没有充分意识到去重规则应该写进 Methods，也没有主动把字段质量和后续图表可靠性联系起来。

此外，我在后期项目整合中的参与较少。README、参数说明、筛选记录和 reflection 等材料主要由高俊杰补齐。以后我应该更主动地把自己负责的数据清洗过程整理成文档，而不是只完成脚本或表格。

## 4. 未来可以学习的内容

后续我希望继续学习：

1. Pandas 数据清洗的更规范写法。
2. 如何处理作者、机构、关键词等多值字段。
3. 如何建立 field dictionary，让别人理解每个字段的含义。
4. 如何记录 screening records，说明每一步筛选数量和理由。
5. 如何把数据质量分析写进论文 Methods 部分。

## 5. 总结

这次项目让我认识到，数据清洗不是项目中的辅助工作，而是所有计量分析可信度的基础。我主要参与了 M1 阶段的数据清洗和质量检查，但后续还需要提高文档意识和方法解释能力。
