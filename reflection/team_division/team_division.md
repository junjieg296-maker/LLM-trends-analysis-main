# 小组分工与贡献说明

## 1. 项目整体说明

本项目主题为 **基于文献计量学的 LLM 多智能体协作研究趋势分析**。项目最终形成了一个可提交的 GitHub 仓库，包含数据、代码、图表、阶段报告、mini review、答辩材料、AI 使用说明、参数说明和个人反思等材料。

从实际完成过程看，本项目虽然是小组合作完成，但主要的项目统筹、工程结构搭建、代码整合、README 完善、图表逻辑修订、最终报告整理、答辩材料归档和 GitHub 巡检材料补齐工作主要由 **高俊杰** 完成。其他组员分别参与了检索、清洗、图谱和综述等模块，为项目提供了必要的阶段性支持。

## 2. 总体分工原则

本项目按照课程项目制要求，采用“数据获取 - 数据清洗 - 计量分析 - 图谱构建 - 综述写作 - GitHub 归档”的流程推进。由于最终提交需要保证仓库结构完整、材料可复查、图表和 RQ 对应清楚、README 能说明项目全貌，因此后期大量整合工作集中在高俊杰身上。

整体贡献可以概括为：

- **高俊杰：承担主要工作与最终责任。** 负责项目主线、仓库结构、代码复现、README、报告整合、答辩材料、reflection、AI 使用说明、参数说明、3 图 1 表检查等大部分最终提交工作。
- **赵世铎：辅助检索与数据获取。** 主要参与 Lens.org 检索口径和原始数据获取。
- **解明昊：辅助数据清洗与质量检查。** 主要参与去重、字段质量和 M1 数据报告相关内容。
- **杨广宸：辅助计量分析与图谱生成。** 主要参与 M2 图表、网络分析和部分指标结果。
- **罗博伟：辅助文献筛选与综述写作。** 主要参与代表文献梳理、综述文字和技术主线讨论。

## 4. 高俊杰承担的核心工作

### 4.1 项目整体统筹

高俊杰负责把课程要求转化为项目任务清单，并在 M1、M2、M3 不同阶段持续检查项目是否满足要求。尤其在后期，课程对 GitHub 仓库结构、README、3 图 1 表、AI 使用说明、个人反思和答辩材料都有明确要求，高俊杰负责逐项补齐。

### 4.2 GitHub 仓库工程化

高俊杰负责将分散的脚本、报告、图表和论文整理为相对规范的仓库结构，包括：

- `config/`：检索式配置。
- `data/`：原始数据、清洗数据、字段字典、筛选记录。
- `src/`：分析脚本。
- `outputs/`：图表和结果输出。
- `reports/`：M1-M3 阶段报告。
- `paper/`：mini review 终稿。
- `presentation/`：答辩材料。
- `reflection/`：小组分工和个人反思。

### 4.3 可复现流程整理

高俊杰负责整合 `run_pipeline.py` 和 `requirements.txt`，使项目能够通过统一命令复现主要分析流程。相比单独提交图片，这一步让项目更符合课程中“可复现 GitHub 项目”的要求。

### 4.4 README 重写与证据链补强

高俊杰多次修改 README，补充以下内容：

- 研究主题和项目简介。
- GitHub 巡检材料导航。
- 小组成员分工。
- RQ 与图表证据对应。
- 课程要求版 3 图 1 表。
- Table 1 Top 10 高被引文献表。
- 数据来源、筛选流程和字段质量。
- M1-M3 阶段成果。
- 图表索引、复现命令、局限性和课程交付物。

### 4.5 图表逻辑和综述结构修订

高俊杰负责检查图表是否真的服务于 RQ，避免出现“图做了但不知道说明什么”的问题。尤其对以下逻辑进行了整理：

- 年度发文图用于回答发展阶段。
- 作者合作图用于回答合作格局。
- 关键词共现图用于回答研究热点。
- 高被引文献表和共被引网络用于回答知识基础。
- 挑战议程图用于回答研究空白和未来方向。

### 4.6 最终材料补齐

根据 Lesson 27 & 28 的巡检要求，高俊杰补齐了：

- `docs/ai_usage.md`
- `params.md`
