# 小组分工与贡献说明

## 1. 项目整体说明

本项目主题为 **基于文献计量学的 LLM 多智能体协作研究趋势分析**。项目最终以 GitHub 仓库形式提交，包含检索式、原始数据、清洗数据、分析脚本、可视化结果、M1-M3 阶段报告、mini review、答辩材料、AI 使用说明、参数说明和个人反思等内容。

在实际推进过程中，小组采用“前期资料收集 + 中期数据分析 + 后期成果整合”的方式合作。由于课程最终提交强调 GitHub 仓库完整性、README 可读性、图表与 RQ 的对应关系、3 图 1 表、个人反思和可复现说明，因此项目后期需要大量整理、重写、检查和补齐工作。高俊杰在这些环节中承担了主要工作，其他成员分别参与检索、清洗、图谱和综述等模块，为项目提供基础材料和阶段性支持。

## 2. 分工原则

本项目的分工不是简单按照“每人写一小段”划分，而是按照文献计量项目的完整流程拆分：

1. 明确研究主题和检索对象。
2. 获取 Lens.org 数据并记录检索策略。
3. 清洗数据并进行质量检查。
4. 完成年度趋势、关键词、合作网络、引用和共被引分析。
5. 从计量结果中提炼综述问题和研究发现。
6. 写作 mini review，并把图表放入合理的解释链条。
7. 整理 GitHub 仓库，使其能够被老师快速巡检和复现。

其中，高俊杰负责贯穿全流程的统筹、整合和最终提交工作，尤其负责把分散的代码、图表、报告和文字材料整理成一个完整项目。其他组员的工作主要集中在某些具体模块，为后续分析和写作提供支撑。
