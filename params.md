# 参数与阈值说明

本文档记录本项目文献计量分析中的关键参数、选择理由和对应输出，用于满足课程对 Methods 透明度和 GitHub 巡检中“参数要写选择理由”的要求。

## 1. 数据检索参数

| 参数 | 设置 | 选择理由 |
|---|---|---|
| 数据库 | Lens.org Scholarly Works | 覆盖学术论文、会议论文、DOI、引用和开放获取等字段，适合课程项目复现 |
| 时间范围 | 2020-2026 | 覆盖 LLM 和 agentic AI 快速发展阶段，兼顾早期探索与 2024-2025 爆发期 |
| 文献类型 | Article, Review, Conference Paper | 保留期刊论文、综述和会议论文，适合 AI / 多智能体领域的发表结构 |
| 语言 | English | 降低跨语言字段标准化复杂度，保证关键词和摘要分析一致性 |
| 检索逻辑 | `(Object) AND (Method)` | Object 锁定 LLM / Generative AI，Method 锁定 multi-agent / collaboration / coordination |

详细检索式见 `config/query.yaml`。

## 2. 数据清洗参数

| 参数 | 设置 | 选择理由 |
|---|---|---|
| 原始记录数 | 542 | Lens.org 初始导出结果 |
| 去重字段 1 | DOI | DOI 是学术文献去重的优先标识 |
| 去重字段 2 | Title | 用于处理 DOI 缺失或 DOI 字段不稳定的重复记录 |
| 清洗后记录数 | 522 | 去除 20 条重复记录后得到有效样本 |
| 缺失值策略 | 保留记录，统计字段覆盖率 | 避免因个别字段缺失过度删除样本，同时在解释中说明边界 |

## 3. 关键词共现参数

| 参数 | 设置 | 选择理由 |
|---|---|---|
| 分析单位 | 文献关键词 | 关键词能直接反映研究主题 |
| 单篇内去重 | 是 | 避免同一篇文献重复关键词造成自连边或虚高共现 |
| 共现阈值 | >= 2 | 敏感性分析显示阈值 2 能在保留信息量和过滤噪声之间取得平衡 |
| 输出图表 | `outputs/figures/keyword_cooccurrence_network.png` | 用于回答 RQ3 研究热点和主题结构 |

关键词阈值敏感性结果：

| 阈值 | 边数 | 节点数 |
|---:|---:|---:|
| 1 | 19,681 | 1,117 |
| 2 | 2,511 | 233 |
| 3 | 948 | 195 |
| 4 | 439 | 189 |
| 5 | 71 | 25 |
| 10 | 7 | 5 |

## 4. 作者合作网络参数

| 参数 | 设置 | 选择理由 |
|---|---|---|
| 分析单位 | 作者 |
| 边定义 | 同一篇文献中共同署名 |
| 原始网络 | 2,502 个节点，12,227 条边 | 展示完整合作关系 |
| 展示阈值 | 合作次数 >= 2 | 过滤一次性合作，突出稳定合作团队 |
| 过滤后网络 | 51 个节点，48 条边 | 用于说明研究合作仍较分散 |

## 5. 引用与共被引参数

| 分析 | 设置 | 选择理由 |
|---|---|---|
| 高被引论文 | 按 Lens.org Citing Works Count 排序取 Top 10 | 用于初步识别 milestone 候选论文 |
| 引用网络 | 基于参考文献字段构建 | 用于观察知识传播路径和高影响节点 |
| 共被引网络 | 统计文献被共同引用关系 | 用于识别共同知识基础和主题簇 |
| References 覆盖率 | 81.99% | 足以开展初步引用分析，但需要说明字段缺失边界 |

## 6. 时间窗口敏感性参数

| 时间窗口 | 文献数 | 篇均被引 | h 指数 |
|---|---:|---:|---:|
| 2020-2023 | 28 | 36.07 | 14 |
| 2021-2024 | 108 | 26.19 | 28 |
| 2022-2025 | 450 | 9.29 | 31 |
| 2023-2026 | 513 | 8.04 | 30 |
| 2020-2026 | 522 | 8.17 | 31 |

解释：全时间窗口的 h 指数为 31，与 2022-2025 窗口一致，说明核心影响力判断较稳定。2026 年数据尚未完整，因此解释趋势时不把 2026 与完整年份直接比较。
