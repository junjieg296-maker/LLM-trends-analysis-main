本项目严格落实“项目制·可复现”的核心学术要求，绝不使用“随便试出来”的参数或黑盒操作。我们旨在通过对**大语言模型多智能体协作 (LLM Multi-Agent Collaboration)** 领域的学术文献进行系统性的检索、清洗、计量分析与知识图谱可视化，精准刻画该领域的技术演化脉络，识别出改变领域走向的里程碑论文，并在此基础上发掘未来的研究空白。

## 📁 规范化目录结构

项目在文件管理上严格分离了“原始数据”、“处理逻辑”与“分析产出”，确立了单一事实来源 (Single Source of Truth)：

* `config/`: 存放参数化配置文件。包含不断迭代的检索式配置 `query.yaml (v0.1)`。
* `data/raw/`: 存放从 Lens.org 导出的 542 篇原始文献 CSV。**(受 .gitignore 保护，不上云)**
* `data/processed/`: 存放经代码去重后的 522 篇高标准数据集，作为后续所有计量的唯一数据源。
* `src/`: 存放 Python 源代码。如 `data_quality.py` 自动化去重与质检脚本。
* `outputs/figures/`: 存放生成的各类高质量图表，如 `publication_trend.png` 和 `wordcloud_titles.png`。
* `reports/`: 存放阶段性验证文档，如采用 PRISMA 规范编写的 `data_quality.md`。
* `paper/`: 存放最终撰写的学术综述稿件初稿与定稿。
* `docs/`: 存放项目先期定义文档，如 M2 阶段知识图谱数据模型设计草图。
* `CHANGELOG.md`: 项目核心里程碑与检索策略变更记录。

## 📅 项目核心里程碑 (Milestones)

项目工作流完全对齐课程进度，拆解为三个关键验收节点：

### [x] M1阶段：数据与检索方案验证（第 4 周末前）
* **阶段目标**：跑通从“初步检索”到“数据清洗”再到“文献筛选”的完整数据处理链条。
* **核心产出**：
    * 确立了 v0.1 版本的 YAML 检索配置文件，完成 Lens.org 数据采集。
    * 输出 `reports/data_quality.md`，包含 PRISMA 流程与字段完整度自检表格。
    * 生成发文趋势图与词云，初步形成领域前沿洞察报告。

### [ ] M2阶段：计量分析与图谱产出（第 10 周末前）
* **阶段目标**：基于 522 篇干净数据集，完成文献网络构建与结果可视化输出。
* **核心产出**：
    * 逻辑严密的图数据模型草图（定义节点、连边及权重属性）。
    * 作者共现网络、机构合作网络及高频关键词共现图谱 (VOSviewer / Python NetworkX)。
    * 包含计算公式及参数敏感性测试对照记录的计量指标规范文档。

### [ ] M3阶段：终稿与项目归档（第 15 周末前）
* **阶段目标**：整合前期的计量证据链，提交最终的学术结论与规范的开源工程资产。
* **核心产出**：
    * 结构完整、逻辑严密的学术综述（Mini Review）稿件（6-8页）。
    * 代码整理完毕、注释清晰的开源 GitHub 仓库，确保分析图表 100% 可复现。
