# 基于文献计量学的大语言模型 (LLM) 前沿趋势追踪

本项目为《文献计量学、前沿趋势追踪与项目制学习框架》课程的项目制学习产出。

本项目严格落实“项目制·可复现”的核心学术要求，绝不使用“随便试出来”的参数或黑盒操作。我们旨在通过对大语言模型（Large Language Models）领域的学术文献进行系统性的检索、清洗、计量分析与知识图谱可视化，精准刻画该领域的技术演化脉络，识别出改变领域走向的里程碑论文，并在此基础上发掘未来的研究空白。

## 📁 规范化目录结构

项目在文件管理上严格分离了“原始数据”、“处理逻辑”与“分析产出”，具体目录架构及其业务逻辑说明如下：

* `config/`: 存放所有参数化的配置文件。主要包括检索式配置文件及不断迭代的同义词词表，确保检索步骤完全可复刻。
* `data/raw/`: 存放从学术数据库直接导出的原始文献数据集。**注意：该目录受版本控制保护，绝对不会上传至公开仓库，以避免数据冗余和隐私合规风险。**
* `data/processed/`: 存放经过代码自动化去重、字段补全和人工消歧后的高标准、干净的数据集，作为后续所有计量分析的唯一数据源。
* `src/`: 存放所有的 Python 源代码文件。包括但不限于数据加载脚本、质量检查脚本、指标计算函数等。
* `outputs/`: 存放代码运行生成的各类结果。包括数据统计表格文件，以及内部子文件夹 `figures/` 中存放的所有高质量网络可视化图表。
* `reports/`: 存放各阶段的过程控制与合规自检文档。如数据质量核查报告、检索式变更日志、查新对比报告、计量指标规范定义表等。
* `paper/`: 存放最终撰写的学术综述稿件初稿与定稿。
* `docs/`: 存放项目先期定义与规则说明类文档，例如项目知识图谱的数据模型设计草图。

## 📅 项目核心里程碑 (Milestones)

项目工作流完全对齐课程进度，拆解为三个关键验收节点：

### M1阶段：数据与检索方案验证（第 4 周末前）
* **阶段目标**：成功跑通从“初步检索”到“数据清洗”再到“文献筛选”的完整数据处理链条。
* **核心验收物清单**：
    * 明确且记录了迭代原因的检索配置文件（YAML格式）及变更日志。
    * 包含各项字段缺失率和重复率精准统计的数据质量自检报告。
    * 具备清晰准入/排除标准、包含剔除原因代码的 PRISMA 文献筛选流程报告。
    * 初步的研究查新与研究空白分析文档。

### M2阶段：计量分析与图谱产出（第 10 周末前）
* **阶段目标**：基于干净的数据集，完成各类文献网络的构建、核心计量指标的计算与结果可视化输出。
* **核心验收物清单**：
    * 逻辑严密的图数据模型草图（清晰定义节点、连边及权重属性）。
    * 包含计算公式、适用分析场景和潜在局限性的计量指标规范文档。
    * 核心知识图谱、作者/机构合作网络图，以及支撑图表稳定性的参数敏感性对照记录。

### M3阶段：终稿与项目归档（第 15 周末前）
* **阶段目标**：整合前期的计量证据链，提交最终的学术结论与规范的开源工程资产。
* **核心验收物清单**：
    * 结构完整、逻辑严密、论证充分的学术综述（mini review）稿件（6-8页）。
    * 代码整理完毕、注释清晰的开源仓库，确保任何人获取代码后均能复现相同的分析图表。
